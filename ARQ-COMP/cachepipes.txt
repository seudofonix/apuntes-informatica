Normalmente los datos que se guardan a memoria cache se cargan in bulk y dentro del rango leido de la ram esta el dato que buscamos. Realmente no cuesta mucho mas cargar por bulk, lo mas caro es no encontrar el dato en tiempo de ejecucion. 

La direccion apunta a un solo byte pero esta particionada para identificar que hay en cache. Para que la cache pueda obtener la respuesta de si tiene el dato se parte el dato en partes. INDICE/ETIQUETA/DESPLAZAMIENTO



Bloques de memoria y lineas de cache.
La cache no guarda bytes sueltos: guarda bloques completos de memoria. 

La memoria principal se parte en bloques: bloques de 16 bytes por ej, cache de 64 lineas de 1 via. (correspondencia directa) La direccion 1234 pertenece al bloque 77.

Cada linea se corresponde a un bloque. Para una direccion concreta: direccion 1234 / 16 bytes por bloque. Bloque = 1234 div 16 = 77. Desplazamiento = 1234 mod 16 = 2; 

Un rango esta entre 2 y 4 lineas para cada bloque. Que linea se usa por cada bloque en funcion del numero del bloque. Para saber en que linea se guarda se guarda en orden. Once we get to the end of the cache memory we loop back to the beginning. EACH RAM ADDRESS HAS A PREDETERMINED CACHE INDEX. 

como los bloques son de 16 bytes, y la cache de 64 bytes de 1 via

-----------------------------------------------
direccion 1234 / 16 bytes por bloque
bloque = 1234 div 16 = 77
desplazamiento = 1234 mod 16 = 2

etiqueta = (1234 div 16) div 64 = 1
Lineo (conjunto) = (1234 div 16) mod 64 = 13
----------------------------------------------

Cuantos conjuntos hay? Lineas totales / vias = 128 / 4 = 32 conjuntos.

1) etiqueta
direccion de 20 bits

EJ3: CALCULO DE CPI

Los fallos que pueden ocurrir accediendo a cache.

Un procesador tiene CPI base = 1,2 si no hay fallos de cache.
L1 tiene tasa de acientos del 92%.  Cada fallo de L1 agrega 8 ciclos para constultar L2.

El sistema completo L1L2 logra 98,5% de aciertos; por lo tanto, 1,5% de los accessos van a memoria princupal, con penalizacion de 70 ciclos. 
Calcular CPI efectivo y una cache de un nivel equivalente.



CPI efectivo = CPI base + ciclos extra por fallos L1 + ciclos exttra por fallo de L2 (acceso a memoria principal).



CPI efectivo

CPI = 1,2 + 0,08 x 8 + 0,015 x 70
CPI = 1,2 + 0,64 + 1,05 = 2,89

El procesador no pasa de 1,2 a 2,89 por ejecutar instrucciones mas complejas, sino por ciclos de espera causadaos por fallos de cache.


Si una cache de un solo nivel tiene la misma penalizacion de memoria de 70 ciclos, debe cumplir.


CPI BASE: 1,2
CPI efectivo: 2,89
Hit equivalente: 97,6
la jerarquia reduce miss penalty.



un hit en l1 no agrega penalizacion; se ejecuta con el cpi base

un mis en l1 no siempre lleg a a memoria principal; primero puede resolverse en l2,.

por eso distinguimos entre fallo de L1 y fallo total del sistema L1+L2.
la penalizacion se pondera por probabilidad: un evento raro pero caro puede dominar el CPI.
