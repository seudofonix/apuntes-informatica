E/S en RISCV

Como se comunica un programa RISC-V con dispositivos externos usando direcciones especiales y polling?

En E/S mapeada a memoria, algunas direcciones no son RAM comun: al leerlas o escribirlas se interactua con un dispositivo. 
There's an I/O module that we can write to, kind of how it works in linux where different devices are treated as files by the filesystem.

(MMIO): Reserved addresses for device registries. One SW might turn on a display or send a character.

Polling: El programa pregunta repetidamente si el dpispositivo esta listo antes de leer o escribir.

Memoria comun: Direcciones donde el programa guarda variables y datos. Buffer, contador, cadena.


CPU sends and receives data using data and control addresses. 

Estado / ready: indica si hay dato disponible o si el display puede recibir: se lee con lw
datos: contiene el caracter, numero o patron que se transfiere: se lee con lw o se escribe con sw
control: configura o habilita un comportamiento del dispositivo: se escribe co nsw.
