There's several ways to implement a stack. Either using an array or a list. These implementations have limitations, size most importantly.
Though an array allows you to read it much more easily which lists don't, but since we're doing stacks anyway does it really matter..?


ARRAY STACKS
fixed max size, much more memory efficient. Random access speed matters. Simpler implementation less code to debug.

LINKED STACKS
Size is unpredictable or highly variable
You can't afford resizing costs or you have to resize very often.
Frequent push/pop of large elements. (You're just moving pointers, not copying data)
Memory is fragmented, you don't need a large block of memory.
Worse cache locality.

Known max size -> array stack
highly variable size -> linked stack
